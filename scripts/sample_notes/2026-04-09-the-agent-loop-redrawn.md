# The agent loop, redrawn

I keep redrawing this. The agent loop is not a loop. It's a tree. Each branch is a hypothesis, and the agent prunes branches that didn't pan out.

If you design for the tree, the UI gets easier. You show the user the branches they didn't take. They learn from the pruning. That's the actual product — not the answer, the audit trail of how the answer was reached.
